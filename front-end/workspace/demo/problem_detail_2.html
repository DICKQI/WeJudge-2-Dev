2001. 斯捷同学办卡记

题目描述 评测描述 我的提交 评测统计
  • 通过次数： 20
  • 提交次数： 100
  • 题目难度： 一般
  • 题目来源： 北京师范大学珠海分校ACM程序设计竞赛协会
斯捷小同学在丢了几个月的卡后,在宿管阿姨的唠唠叨叨下总算下定决心去补办校园卡了。一大清早斯捷这个小伙子就跑去农行了，结果农行门都还没开呢，门口就排了一大群人了。
办好卡的斯捷同学坐在一旁，闲得蛋疼的他决定抱着电脑边敲代码边记录来农行办业务的人以及他们的行为。
来农行办业务的人一共有三种行为:
  1. 安安心心、踏踏实实排队的小伙子，直到业务办完才会离开
  2. 不安安心心、不踏踏实实的小伙子，排队还没轮到他就跑了(小伙子，你这样可不行啊)
  3. 大大方方、乐于助人的小伙子，就算中途放弃也要把叫号票给别人(票会给除了自己的随机一个人)
请你帮助斯捷同学，打印成功办理业务的人员名单(按叫号顺序输出)

输入要求

每次先输入一个整数T（1 <= T <= 30），接下来T组测试数据
对于每一组数据，第一排有一个整数n（1 <= n <= 10000）表示斯捷同学记录了他在银行这段时间来办理卡的同学们进行的n次操作，接下来n行数据，每一行肯定有两个整数a，b（1 <= a <= 3）（1 <= b <= n），可能还有一个整数c（1 <= c <= n && c != b）。
当a等于1时，表示b同学刚来农行,取了一个号并排队等待
当a等于2时，表示b同学还没办理业务就跑了,此时他原本的号码会报废,并且他不会把号码给别人
当a等于3时，再读入一个整数c表示b同学把号给了c同学并跑路了,此时c同学会将两张号中较大的号扔进垃圾桶(当然没有人会去翻垃圾桶的,所以不会有人捡到他扔掉的号码)  

输出要求

每次输出占一行，按叫号顺序输出成功办理业务的人员名单,每个数字后面都有空格

输入示例

2
6
1 1
1 2
1 3
2 3
1 4
3 1 4
6
1 1
1 2
1 3
1 4
3 4 1
2 3

输出示例

4 2
1 2
